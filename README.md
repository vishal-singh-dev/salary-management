# Salary Management

Salary Management is a full-stack HR application for managing employee profiles, salary records,
and compensation analytics for a 10,000-employee demo dataset.

The project includes a FastAPI backend, PostgreSQL schema and migrations, deterministic seed data,
and a Next.js frontend for employee management and salary insights.

## Why This Project Is Useful

- Manage employee profiles with create, view, update, and soft-delete workflows.
- Store current salary structures with base salary, variable pay, HRA, PF, and gratuity amounts.
- Analyze salary distribution by country, department, and job title.
- Use fixed master data for country, department, and job-title dropdowns.
- Bootstrap a complete local demo environment with Docker Compose.
- Validate backend behavior with pytest and frontend behavior with Vitest.

## Quick Start With Docker

Docker Compose is the fastest way to run the full project from a fresh clone.

```powershell
git clone https://github.com/vishal-singh-dev/SalaryManagement.git
cd SalaryManagement
docker compose up --build
```

Open:

```text
Frontend: http://localhost:3000
Backend:  http://localhost:8000
API docs: http://localhost:8000/docs
```

The backend container automatically:

- waits for PostgreSQL to be healthy,
- runs Alembic migrations,
- seeds fixed exchange rates,
- seeds filter master data,
- creates the 10,000-employee demo salary dataset when the employee table is empty.

Stop the stack:

```powershell
docker compose down
```

Reset the local Docker database:

```powershell
docker compose down -v
```

Seed volume and randomness can be changed in `docker-compose.yml`:

```yaml
SEED_EMPLOYEE_COUNT: "10000"
SEED_RANDOM_SEED: "2026"
SEED_BATCH_SIZE: "1000"
```

## Manual Backend Setup

Install prerequisites:

- Python 3.12+
- uv
- PostgreSQL or a Neon PostgreSQL database

From the backend folder:

```powershell
cd backend
uv sync --dev
Copy-Item .env.example .env
```

Set `backend/.env`:

```dotenv
ENVIRONMENT=development
DATABASE_URL=postgresql://username:password@host/database
CORS_ORIGINS=["http://localhost:3000"]
SQL_ECHO=false
```

Run migrations and seed data:

```powershell
uv run alembic upgrade head
uv run salary-seed-all --count 10000 --random-seed 2026
```

Start the API:

```powershell
uv run uvicorn app.main:app --reload
```

Verify:

```powershell
curl http://127.0.0.1:8000/health
```

## Manual Frontend Setup

Install prerequisites:

- Node.js 22+
- npm

From the frontend folder:

```powershell
cd frontend
npm install
Copy-Item .env.example .env
```

Set `frontend/.env`:

```dotenv
PUBLIC_API_BASE_URL=http://127.0.0.1:8000
```

Start the frontend:

```powershell
npm run dev
```

Open:

```text
http://localhost:3000
```

## Usage

Main frontend routes:

| Route | Purpose |
| --- | --- |
| `/dashboard` | Salary analytics dashboard |
| `/employees` | Employee list with pagination |
| `/employees/new` | Create employee with initial salary |
| `/employees/{id}` | View and update employee profile |
| `/employees/{id}/salary` | View salary history table |

Backend API documentation is generated by FastAPI:

```text
http://localhost:8000/docs
```

Useful API checks:

```text
GET /health
GET /api/v1/employees?limit=25&offset=0
GET /api/v1/master-data
GET /api/v1/analytics/salaries?country_code=IN&department=ENG
GET /api/v1/analytics/salaries?currency_basis=usd&department=ENG
```

## Data Model

The backend stores normalized PostgreSQL data:

| Table | Purpose |
| --- | --- |
| `employees` | Employee profile, master-data values, country code, and employment period |
| `employee_salary_records` | Effective-dated salary structure |
| `exchange_rates` | Fixed currency-to-USD rates for salary comparison |
| `master_data` | Country, department, and job-title dropdown values |
| `audit_events` | Audit trail for create, update, delete, and seed events |

Employee `department` and `title` store master-data `value` codes. The frontend displays the
matching master-data `description`.

## Deployment

The project is Docker-ready and can be deployed to AWS with the same containers used locally.
A straightforward path is:

1. Provision a PostgreSQL database with Amazon RDS or use an external PostgreSQL provider.
2. Build and run the backend container with the production database URL.
3. Build and run the frontend container with the public backend URL.
4. Put the backend and frontend behind HTTPS using an AWS load balancer, reverse proxy, or managed
   container service.

Backend environment variables:

```text
DATABASE_URL=<your PostgreSQL connection string>
ENVIRONMENT=production
CORS_ORIGINS=["https://your-frontend-domain"]
SQL_ECHO=false
```

Frontend environment variable:

```text
PUBLIC_API_BASE_URL=https://your-backend-domain
```

The backend Docker startup runs migrations and bootstraps fixed seed data before starting the API.
Employee demo data is created only when the employee table is empty.
